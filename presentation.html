How It Works

Four steps, fully automated

1

Connect your accounts

GitHub via VS Code's built-in OAuth. No passwords stored. Render via API key in SecretStorage.

2

Auto-generate render.yaml

Samosa detects your stack and writes the deployment config automatically. Supports static HTML/CSS/JS, Node.js, Python, Go and React.

3

Push to GitHub

Creates a new repo (or uses existing), commits and pushes via git and the GitHub API.

4

Trigger Render deployment

Calls the Render API to create and deploy the service. Your app is live.

Zero Config

We write the config for you

render.yaml is a config file that tells Render how to build and run your project. Samosa generates it for you and commits it automatically.

# render.yaml: auto-generated by Samosa
services:
  - type: web
    name: my-project
    env: node
    buildCommand: npm install
    startCommand: node index.js
    plan: free
Tech Stack

Built on top of VS Code

No external server. All auth is local. Packaged as a standard .vsix extension.

🧩 VS Code API
🐙 Octokit / GitHub API
🌐 Render API
🔐 SecretStorage (API key)
🔑 vscode.authentication (OAuth)
⚡ esbuild
🔧 child_process (git)
